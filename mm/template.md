# Software Requirements Specification

## 1. Overview
Purpose, scope, business context, and a summary of the proposed solution in Odoo.

## 2. Actors
The user roles / personas and external systems that interact with the solution.

## 3. Functional Requirements
For each requirement: ID, title, description, type, MoSCoW priority, target Odoo module, actors,
acceptance criteria, dependencies, and source references.

## 4. Non-Functional Requirements
Performance, security, usability, availability, data retention, localization, etc. — with measurable
metrics wherever possible.

## 5. Glossary
Domain and Odoo terms used in this document.

## 6. Open Questions
Anything ambiguous or missing from the source material that needs client clarification.

## 7. Diagrams
Mermaid diagrams that illustrate the system:
- **System context / high-level flow** — how users, Odoo, and external systems interact.
- **Data model (ER)** — the main entities and their relationships.
- **Key business workflows** — the important end-to-end processes.

Each diagram has a stable ID (`DGM-001`, …), a title, and a short explanation.
